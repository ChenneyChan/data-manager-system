# 温升测试数据库重构方案

## 概述

将温升测试的温度字段从**固定列模型**重构为**动态字段模型**，使用户可以按产品（workflow）和冷却方式（coolingMode）自定义温度通道，同时保留老数据完整可用。

### 设计决策

| 方案 | 说明 |
|---|---|
| JSON 列存储动态温度 | 一行一个时间点，temps 用 JSON 序列化，无需 JOIN，批量写入简单 |
| 固定列保留电气/水冷字段 | 电压、电流、功率、频率、水冷温度等不变字段保持独立列，历史兼容 |
| 通道定义独立表 | 按 `(workflow_id, cooling_mode)` 唯一键存储，支持每个产品每种冷却方式独立定义 |
| 老表不动 | `commonTempRiseTestRecord` 保留，新老数据共存，渐进迁移 |

## 数据库表结构

### 1. 新增表：`tempRiseChannelDefinition` — 通道定义表

每个产品 + 冷却方式可定义一套独立的温度通道列表。

```sql
CREATE TABLE IF NOT EXISTS tempRiseChannelDefinition (
    id           BIGINT PRIMARY KEY AUTO_INCREMENT,
    workflow_id  VARCHAR(128) NOT NULL COMMENT '产品工作令号',
    cooling_mode VARCHAR(32)  NOT NULL COMMENT '冷却方式: AF / AF+ / AFWF',
    channel_defs JSON         NOT NULL COMMENT '通道定义JSON数组',
    updated_at   DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_workflow_cooling (workflow_id, cooling_mode)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='温升测试通道定义表';
```

**`channel_defs` JSON 结构：**

```json
[
  {"index": 0, "roleKey": "WindingA",  "roleName": "绕组A",     "title": "高压A相",   "probe": "Slot-3"},
  {"index": 1, "roleKey": "WindingB",  "roleName": "绕组B",     "title": "高压B相",   "probe": "Slot-5"},
  {"index": 2, "roleKey": "WindingC",  "roleName": "绕组C",     "title": "高压C相",   "probe": "Slot-7"},
  {"index": 3, "roleKey": "Core",      "roleName": "铁心",       "title": "铁心温度",   "probe": "Slot-9"},
  {"index": 4, "roleKey": "Custom1",   "roleName": "油箱顶部",   "title": "油箱顶部",   "probe": "Slot-11"}
]
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `index` | int | 通道序号（0-based），决定 DataGrid 列顺序 |
| `roleKey` | string | 内部标识，用于字段匹配（如 `WindingA`） |
| `roleName` | string | 显示名称，回退值 |
| `title` | string | 用户自定义标题，UI 各处显示用 |
| `probe` | string | 物理探头槽位号（如 `Slot-3`） |

---

### 2. 新增表：`tempRiseTestRecord` — 温升数据记录表

替代 `commonTempRiseTestRecord`，固定字段 + JSON 动态温度。

```sql
CREATE TABLE IF NOT EXISTS tempRiseTestRecord (
    id                    BIGINT PRIMARY KEY AUTO_INCREMENT,
    test_info_id          VARCHAR(36)  NOT NULL COMMENT '关联 commonTempRiseConfigRecord.ID',
    workflow_id           VARCHAR(128) NOT NULL COMMENT '产品工作令号',
    timestamp             DATETIME     NOT NULL COMMENT '采集时间点',
    -- 固定字段: 电压/电流/功率/频率
    ua FLOAT DEFAULT NULL COMMENT 'A相电压(V)',
    ub FLOAT DEFAULT NULL COMMENT 'B相电压(V)',
    uc FLOAT DEFAULT NULL COMMENT 'C相电压(V)',
    u3 FLOAT DEFAULT NULL COMMENT '三相平均电压(V)',
    ia FLOAT DEFAULT NULL COMMENT 'A相电流(A)',
    ib FLOAT DEFAULT NULL COMMENT 'B相电流(A)',
    ic FLOAT DEFAULT NULL COMMENT 'C相电流(A)',
    i3 FLOAT DEFAULT NULL COMMENT '三相平均电流(A)',
    p3 FLOAT DEFAULT NULL COMMENT '三相总有功功率(kW)',
    fu FLOAT DEFAULT NULL COMMENT '频率(Hz)',
    -- 动态字段: 用户自定义温度通道
    temperatures          JSON         NOT NULL COMMENT '温度通道值',
    -- 固定字段: 水冷设备 (仅 AFWF 模式有值)
    outlet_water_temp     FLOAT DEFAULT NULL COMMENT '出水口温度(℃)',
    inlet_water_temp      FLOAT DEFAULT NULL COMMENT '回水口温度(℃)',
    ambient_temp_1        FLOAT DEFAULT NULL COMMENT '环境温度1(℃)',
    ambient_temp_2        FLOAT DEFAULT NULL COMMENT '环境温度2(℃)',
    outlet_air_temp_1     FLOAT DEFAULT NULL COMMENT '出风口温度1(℃)',
    outlet_air_temp_2     FLOAT DEFAULT NULL COMMENT '出风口温度2(℃)',
    outlet_air_temp_3     FLOAT DEFAULT NULL COMMENT '出风口温度3(℃)',
    outlet_air_temp_4     FLOAT DEFAULT NULL COMMENT '出风口温度4(℃)',
    outlet_air_temp_5     FLOAT DEFAULT NULL COMMENT '出风口温度5(℃)',
    outlet_air_temp_6     FLOAT DEFAULT NULL COMMENT '出风口温度6(℃)',
    outlet_air_temp_7     FLOAT DEFAULT NULL COMMENT '出风口温度7(℃)',
    outlet_air_temp_8     FLOAT DEFAULT NULL COMMENT '出风口温度8(℃)',
    water_flow_rate       FLOAT DEFAULT NULL COMMENT '水流量(m³/h)',
    INDEX idx_workflow_time (workflow_id, timestamp),
    INDEX idx_test_info (test_info_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='温升测试数据记录表';
```

**`temperatures` JSON 结构：**

```json
{
  "WindingA": 23.5,
  "WindingB": 24.1,
  "WindingC": 23.8,
  "Core": 30.2,
  "EnvA": 18.0,
  "Custom1": 22.3
}
```

Key 为通道的 `roleKey`，Value 为 `float` 型温度值。通道集合完全由 `tempRiseChannelDefinition.channel_defs` 定义，无固定列限制。

---

### 3. 保留表（不做修改）

| 表名 | 说明 | 状态 |
|---|---|---|
| `commonTempRiseConfigRecord` | 试验批次头表（工作令、试验阶段、冷却方式等） | **不变**，新老数据共用 |
| `commonTempRiseTestRecord` | 老版固定列温升记录表 | **保留不动**，老数据可继续查询 |
| `TempRiseCommonInfo` | 温升通用信息（试验电压、电流等） | **不变** |

## 数据读取策略（报告系统适配指南）

### 场景 A：读取一次试验的所有温度数据

**新数据：**
```sql
SELECT t.timestamp, t.ua, t.ub, t.uc, t.u3, t.ia, t.ib, t.ic, t.i3, t.p3, t.fu,
       t.temperatures, t.outlet_water_temp, t.inlet_water_temp, t.water_flow_rate
FROM tempRiseTestRecord t
JOIN commonTempRiseConfigRecord c ON t.test_info_id = c.ID
WHERE c.workflow_id = @workflowId
  AND c.testing_phase = @phase
  AND c.testing_index = @index
ORDER BY t.timestamp;
```

**老数据：**
```sql
SELECT timestamp, ua, ub, uc, u3, ia, ib, ic, i3, p3, fu,
       WindingATemperature, WindingBTemperature, WindingCTemperature,
       CoreTemperature, AmbientATemperature, AmbientBTemperature,
       AmbientCTemperature, AmbientDTemperature, Outlet1, Outlet2, Outlet3,
       Outlet4, Outlet5, Outlet6, Inlet1, Inlet2, Inlet3, TopTemperature,
       OutletWaterTemperature, InletWaterTemperature, WaterFlowRate
FROM commonTempRiseTestRecord
WHERE ID = @configId
ORDER BY timestamp;
```

### 场景 B：获取某次试验的通道定义

**新数据：**
```sql
SELECT channel_defs FROM tempRiseChannelDefinition
WHERE workflow_id = @workflowId AND cooling_mode = @coolingMode;
```
返回的 JSON 数组即该次试验使用的所有通道列表（含 `roleKey`、`title`、`probe` 等）。

**老数据：** 通道定义从本地配置文件 `Configs.TemperatureChannels` 读取，无 DB 对应。

### 场景 C：统一查询新老数据的电气参数

新老表的电气字段列名完全一致（`ua, ub, uc, u3, ia, ib, ic, i3, p3, fu`），可直接 UNION：
```sql
SELECT timestamp, ua, ub, uc, u3, ia, ib, ic, i3, p3, fu
FROM tempRiseTestRecord WHERE test_info_id = @id
UNION ALL
SELECT timestamp, ua, ub, uc, u3, ia, ib, ic, i3, p3, fu
FROM commonTempRiseTestRecord WHERE ID = @id
ORDER BY timestamp;
```

### 场景 D：解析 JSON 温度字段

在新表中，温度数据存在 `temperatures` JSON 列中。不同数据库的提取方式：

**MySQL：**
```sql
SELECT JSON_EXTRACT(temperatures, '$.WindingA') AS WindingA,
       JSON_EXTRACT(temperatures, '$.Core') AS Core
FROM tempRiseTestRecord;
```

**C# 代码中：**
```csharp
var temps = JsonSerializer.Deserialize<Dictionary<string, float?>>(record.TemperaturesJson);
float? windingA = temps.GetValueOrDefault("WindingA");
```

## 数据写入流程（采集系统侧）

```
1. 用户选择/自定义通道 → TempSlotSelectView
2. 点击确认 → 保存到 Configs.Configs.TemperatureChannels (本地JSON文件)
3. 启动采集 → SaveConfig() 调用 SaveChannelDefinitionToDb()
   → 写入 tempRiseChannelDefinition (按 workflow_id + cooling_mode upsert)
4. 采集数据 → DataTable 内存中暂存
5. 上传数据 → UploadDataAsync()
   - 读取 tempRiseChannelDefinition 获取当前通道定义
   - 遍历 SelectedChannels 动态构建 temperatures JSON
   - 写入 tempRiseTestRecord (BatchInsertData)
```

### 通道定义优先级（读取时）

```
数据库 tempRiseChannelDefinition  >  本地 Configs.TemperatureChannels  >  代码默认值 CreateDefaultChannels()
```

页面加载、工作令切换、冷却方式变更时自动从 DB 加载。

## 老数据兼容性

| 场景 | 兼容策略 |
|---|---|
| 老采集数据 | 保留在 `commonTempRiseTestRecord`，表结构未变，所有历史查询不受影响 |
| 老通道配置 | 保留在本地 JSON 配置文件，无 DB 记录时作为默认值 |
| 报告系统 | 查询时判断试验数据在哪个表，分别处理，电气字段列名一致可直接 UNION |
| 采集系统 | 新数据全部写入 `tempRiseTestRecord`，不再写老表 |
